| Вариант | Скорость | Место | Вердикт |
|---------|----------|-------|---------|
| **Полный** | ⭐⭐⭐⭐ | ⭐ | Быстро, но жрёт много места |
| **Инкрементный** | ⭐⭐ | ⭐⭐⭐⭐⭐ | Экономно, но трудно восстанавливать |

### 1.2. Восстановить данные до момента поломки

Всем трём вариантам нужны журналы изменений (логи), чтобы остановиться на нужном времени.

| Вариант | Сложность | Риск | Вердикт |
|---------|-----------|------|---------|
| **Полный** | Средняя | Низкий | ✅ **Лучший вариант** — полный бэкап + журналы |
| **Инкрементный** | Высокая (много цепочек) | Высокий (сломается один — всё) | ❌ Не наш вариант |
| **Дифференциальный** | Средняя | Средний | ⚠️ Середнячок |

### 1.3. Моментальное переключение при поломке

Ни один вид бэкапа не подходит, так как все они требуют времени на восстановление.

**Нужна репликация** — второй сервер-копия, который подхватывает работу за секунды.

## Задание 2. PostgreSQL

### 2.1. Команды резервирования и восстановления

#### Резервирование (создание бэкапа) — утилита `pg_dump`

Проще всего сделать дамп в виде обычного SQL-файла:

```bash
pg_dump -U postgres -d имя_базы > backup.sql
-U — пользователь

-d — название базы данных

> — сохраняем результат в файл

Для больших баз удобнее использовать кастомный формат (сжатый и гибкий):

bash
pg_dump -U postgres -Fc -f backup.dump имя_базы
-Fc — формат custom

-f — имя выходного файла

Восстановление — утилита pg_restore
Если бэкап в SQL-формате, восстанавливаем через psql:

bash
psql -U postgres -d новая_база < backup.sql
Если бэкап в кастомном формате .dump:

bash
pg_restore -U postgres -d новая_база -c backup.dump
-c — очищает базу перед восстановлением

2.1.* Автоматизация резервного копирования
Да, автоматизировать можно — и это стандартная практика.

Самый простой способ — добавить скрипт в Cron
Создаёте bash-скрипт:

bash
#!/bin/bash
# Скрипт: /usr/local/bin/pg_backup.sh

TIMESTAMP=$(date +%Y%m%d_%H%M%S)
pg_dump -U postgres -Fc -f /backups/db_${TIMESTAMP}.dump имя_базы

# Удаляем бэкапы старше 7 дней
find /backups -name "*.dump" -mtime +7 -delete
Добавляете задание в Cron (ежедневно в 2 часа ночи):

bash
0 2 * * * /usr/local/bin/pg_backup.sh >> /var/log/pg_backup.log 2>&1
